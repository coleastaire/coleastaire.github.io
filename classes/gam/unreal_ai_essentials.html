MoveToLocation Tips

  • AcceptanceRadius - distance check for when to perform an action.
  • bStopOnOverlap - stop on overlap of a point, rather than reaching the point from their center.
  • bUsePathFinding - uses NavMesh Option to find target
  • bProjectDestinationToNavigation - projects thre location on thhe navigation data before using it
  • bCanStrafe - whether the AI can traverse diagonally on NavMesh or not
  • FilterClass - allows the use of AreaClass, another nav component.
  • there is also a 'Simple Move To Location' which does similar, except without the ability to modify as many variables
  • you can use the Target Subpoint Actor from Custom Classes as waypoints

Wandering AI Tips

Traces

Made using LineTraceByChannel function, to simulate senses, can trace outward from the character using GetActor<R,L,F,B>Vector and multiplying by a distance.

RandomStream

Created by using the Make RandomStream, with input Initial Seed, which should be the seed, stored, from the last session. You cannot do this with the standard Random Node. You can randomize the seed to simulate true randomness. For seeds, set the max value to 65535.

Path Finding

  • Influence Mapping - info based on location in nav mesh
  • Null Paths - cut areas out of nav mesh
  • Allowed Path - can allow/block AI from an area of the nav mesh

Cell Size: This is the 2D size of the voxel, basically a 3D pixel, which represents a space to navigate. This is the floor area of the voxel. This directly correlates to the resolution of NavMesh. Smaller sizes here generate more voxels, resulting in better movement precision. Unfortunately, performance takes a hit during the generation of NavMesh due to this. Cell Height: This is the height of the voxel. This value correlates to Agent Max Step Height, and it is suggested to be kept at half its value. NavMesh will not be generated if the Cell Height value isnâ€™t at least .1 less than that of Agent Max Step Height. Tile Size UU: When this is at lower values, it increases the NavMesh Generation performance. Try enabling Draw Poly Edges to see this in action. Agent settings (Radius, Height, Max Height, Max Slope, Max Step Height): are specific to your agents and should be specified appropriately. Min Region Area: This gets rid of what looks to be artifacts of pieces of NavMesh Generation, which are too insignificant to navigate.
